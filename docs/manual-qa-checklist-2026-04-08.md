# 수동 QA 체크리스트 (2026-04-08)

## 범위
- `delivery/history` 성격 화면
- `sales/material` 영업자료실
- `master/performance-plan` 성과 계획 관리
- `delivery/plan` 스펙 추가/취소 탭
- `delivery/demand-forecast` 수요 예측
- `partner/notice` 공지/자료실
- `profit/*` 손익분석 상세

## 공통 사전 점검
- [ ] `npm run smoke:pages` 실행 결과가 모두 `OK` 이다.
- [ ] 콘솔 에러/경고가 반복적으로 발생하지 않는다.
- [ ] 한글 라벨/버튼/테이블 헤더가 깨지지 않는다.
- [ ] 버튼 클릭 시 레이아웃이 흔들리거나 요소가 겹치지 않는다.
- [ ] 1366x768, 1920x1080 기준으로 주요 영역이 정상 노출된다.

## 1) 영업자료실 (`/sales/material`)
- [ ] 조회 입력 후 `조회` 동작이 정상이다.
- [ ] 목록 헤더(`제목/유형/등록일/첨부`) 정렬과 폭이 정상이다.
- [ ] 다운로드 버튼 텍스트가 세로 깨짐 없이 가로 정렬로 보인다.
- [ ] 긴 제목 데이터에서도 행 높이/줄바꿈이 깨지지 않는다.
- [ ] `등록` 버튼 클릭 시 등록 플로우가 정상 진입한다.

## 2) 성과 계획 관리 (`/master/performance-plan`)
- [ ] 우측 상단 `저장` 버튼 위치가 조회 버튼 라인과 시각적으로 맞다.
- [ ] 탭(`팀 계획/개인 계획/일괄 업로드`) 전환 시 레이아웃 깨짐이 없다.
- [ ] 테이블 본문 텍스트가 세로 출력되지 않고 가로로 정상 표시된다.
- [ ] 상세 보기 진입 방식이 팝업으로 정상 동작한다.
- [ ] 페이지 우측 여백이 과도하지 않고 컨텐츠 폭이 자연스럽다.

## 3) 납품 계획 관리 (`/delivery/plan`)
- [ ] `스펙 추가` 탭 UI(필터/그리드/버튼)가 기존 스타일과 동일하다.
- [ ] `스펙 취소` 탭 UI(필터/그리드/버튼)가 기존 스타일과 동일하다.
- [ ] 탭 전환 시 데이터/버튼이 사라지지 않는다.
- [ ] 한글 텍스트 깨짐 없이 표시된다.

## 4) 수요 예측 (`/delivery/demand-forecast`)
- [ ] 예측수량을 기존값과 다르게 입력하면 `사유` 입력이 활성화된다.
- [ ] 기존값과 동일하게 되돌리면 `사유` 입력이 비활성화된다.
- [ ] 수정이력 버튼/영역이 노출되고 데이터가 보인다.
- [ ] 저장 후 재조회 시 변경값과 사유가 유지된다(목업 기준 확인).

## 5) 공지/자료실 (`/partner/notice`)
- [ ] 목록/상세 스타일이 프로젝트 공통 톤과 일치한다.
- [ ] 긴 본문 데이터에서 상세 내용 영역 스크롤이 정상 동작한다.
- [ ] 등록 화면 진입/입력/저장 동선이 정상이다.
- [ ] 노출기간/고정여부 등 고도화 필드가 있으면 저장/반영이 정상이다.

## 6) 손익분석 상세 (`/profit/...`)
- [ ] 목록에서 상세 클릭 시 상세 페이지(또는 모달)로 정상 진입한다.
- [ ] 상세 데이터(요약/테이블/차트)가 빈 화면 없이 노출된다.
- [ ] 목업 데이터가 없어도 안내 문구 또는 기본 상태가 정상 출력된다.

## 7) 배포 전 최종 회귀
- [ ] 주요 페이지 이동(사이드 메뉴) 시 404/빈 화면이 없다.
- [ ] 다크/라이트 변형이 있다면 기본 테마에서 가독성 문제가 없다.
- [ ] 저장/조회/초기화 핵심 버튼의 disabled 조건이 의도대로 동작한다.
